# Animation Tab — Logic Reference

Цей файл описує поточну логіку роботи кожного елемента вкладки **Animation**.
Відкоригуй і поверни — реалізую зміни.

---

## 1. Animation (каскадер вибору анімації)

- Показує всі анімації скелету у вигляді каскадного дерева.
- **Вибір анімації** залежить від режиму:
  - **Add mode OFF** → викликає `setAnimation(currentTrack, name, loop)` — замінює поточну анімацію на треку.
  - **Add mode ON** → викликає `addAnimation(currentTrack, name, loop)` — додає анімацію в кінець черги треку.
- Після вибору значення залишається у полі (відображає обраний шлях).
- Якщо вибрати трек в якому вже є анімація — відображає шлях до цієї анімації.
- Після завантаження скелета за замовчуванням усі треки пусті.
- Недоступний поки скелет не завантажений.

---

## 2. Track (сітка 0–11)

- 12 кнопок (0..11) — вибір **активного треку** для подальших дій.
- **Зелений** (active) — поточно вибраний трек.
- **Синій** (running) — на цьому треку є активна анімація в Spine.
- Якщо трек одночасно active + running — колір зелений (active перебиває).
- Недоступні поки скелет не завантажений.

---

## 3. Add mode (кнопка-тогл)

- **OFF (default)** — вибір анімації у каскадері робить `setAnimation` (замінює).
- **ON** — вибір анімації у каскадері робить `addAnimation` (додає в чергу).
- При увімкненому режимі кнопка підсвічується як primary.
- Немає автоматичного вимкнення — лишається ON поки користувач не вимкне.
- Якщо в трек засетити анімацію в режимі OFF, а потім перемкнути на ON — вибір анімації у каскадері робить `addAnimation` (додає в чергу).

---

## 4. Play / Pause / Resume (кнопка)

Один елемент з трьома станами:

| Стан store | Текст кнопки | Дія при натисканні |
|---|---|---|
| `isPlaying = true` | ⏸ Pause | Викликає `pause()` — заморожує на місці |
| `isPlaying = false, isPaused = false` | ▶ Play | Викликає `play()` — перезапускає з початку |
| `isPlaying = false, isPaused = true` | ▶ Resume | Викликає `play()` — відновлює з місця паузи |

**Деталі Play:**
- Якщо `isPaused = false` → Spine-послідовності на кожному треку відновлюються з плейлиста (повний перезапуск).
- Якщо `isPaused = true` → просто знімає глобальний `timeScale = 0`, анімація продовжується з місця зупинки.

**Деталі Pause:**
- Встановлює `timeScale = 0` глобально — усі треки замерзають.
- `isPaused = true` запам'ятовує що це пауза (не стоп).

**Деталі Stop (автоматичний):**
- Спрацьовує коли всі активні треки дійшли до кінця (без loop, без черги).
- Також спрацьовує при `clearTrack` якщо більше немає жодного плейлиста, або при `clearTracks`.

---

## 5. ← 1f / 1f → (кнопки покадрового переходу)

- Переміщують поточну позицію анімації на **активному треку** на ±1/60 секунди (~1 кадр при 60 fps).
- Враховують межі: не виходять за [0, duration].
- Працюють для всіх активних треків одразу.
- Доступні незалежно від стану Play/Pause.

---

## 6. Loop (глобальний тогл)

- Значення `animationStore.loop` — **дефолтне значення loop** при виборі анімації через каскадер.
- Впливає на нові анімації що додаються через каскадер (set або add).
- **НЕ** змінює loop для вже запущених треків (для цього є Loop у блоці треку).
- Не впливає на відтворення безпосередньо.

---

## 7. Speed (слайдер 0–3×)

- Діапазон: 0.0 – 3.0, крок 0.01.
- Мітка `1` на слайдері для зручності.
- Значення відображається справа (`0.75×`, `2.00×` тощо).
- Кнопка `1×` — скидає до 1.0 (недоступна якщо вже 1.0).
- **Вплив:** змінює `state.timeScale` в Spine лише коли `isPlaying = true`.
- Під час паузи зміна speed не впливає на Spine (застосується при Resume).

---

## 8. Active tracks (список активних треків)

Показує всі треки на яких є анімація в Spine.

### 8.1 Заголовок секції

- Лейбл "Active tracks" зліва.
- Кнопка **Clear All** справа — видаляє всі треки та зупиняє відтворення.
  - Недоступна якщо немає активних треків.
  - Скидає всі треки, зупиняє відтворення, робить setToSetupPose для спайну.

### 8.2 Порожній стан

- Текст "No active tracks" коли жоден трек не активний.

### 8.3 Блок треку

Кожен активний трек — окремий блок з:

**Хедер блоку:**
- **Checkbox (enabled)** — увімкнення/вимкнення треку:
  - `true` (default) — трек програється нормально.
  - `false` — якщо loop: трек заморожується (timeScale=0), залишається у списку, але не відтворюється; якщо без loop: наступного разу не буде перезапущений.
- **Loop checkbox** — вмикає/вимикає loop для поточного TrackEntry в Spine.
  - Змінює `entry.loop` напряму, при натисканні Play значення зберігається.
  - При зміні під час playback — негайно застосовується.
- **#N** — індекс треку (інформаційний).
- **✕** — очищає трек (`clearTrack(trackIndex)`):
  - Видаляє плейлист треку зі стору.
  - Викликає `spine.state.clearTrack(N)`.
  - Якщо після цього плейлистів не лишилось — викликає `stop()`.

**Підсвітка блоку:**
- Синя рамка — якщо цей трек є `currentTrack` (вибраний у сітці Track).

**Поточна анімація (▶):**
- Іконка `▶`, назва анімації.
- Прогрес та таймер — **не відображаються в HTML-панелі** (перенесено в Pixi overlay, див. секцію 11).

**Черга анімацій (⏭):**
- Список наступних анімацій у черзі цього треку.
- Іконка `⏭`, назва анімації, кнопка `✕` для видалення.
- `✕` видаляє конкретний запис із черги (`removeQueueEntry(trackIndex, i)`).
  - Видаляє з плейлиста стору та з `TrackEntry.next` у Spine.

## 9. Стан-машина Play/Pause/Stop

```
[STOP]  ──play()──>  [PLAY]  ──pause()──>  [PAUSED]
  ^                     |                      |
  |                     |                      |
  +──stop()─────────────+                      |
  +──stop()────────────────────────────────────+
  ^                     |
  +──auto-stop──────────+  (всі треки досягли кінця без loop/queue)
```

**`isPlaying`** — Spine отримує `timeScale = speed` (або 0 для вимкнених looped).
**`isPaused`** — `isPlaying = false`, але `isPaused = true`, Spine отримує `timeScale = 0`.
**Stop** — `isPlaying = false`, `isPaused = false`, Spine отримує `timeScale = 0`.

---

## 11. Pixi overlay — прогрес треків

Прогрес анімацій відображається **безпосередньо на canvas** (Pixi), а не в HTML-панелі.
Це гарантує 100% синхронізацію з рендером без затримок Vue reactivity.

**Розташування:** лівий нижній кут canvas, поверх сцени.

**Вміст для кожного активного треку** (оновлюється в ticker кожен кадр):
```
#0  walk_cycle      1.23s / 2.40s  [████████░░]  loop
#1  attack          0.45s / 1.20s  [███░░░░░░░]
```

**Правила відображення:**
- Один рядок на трек: `#N  <назва>  <time>s / <duration>s  [прогрес-бар]  [loop]`
- Прогрес-бар — текстовий (символами `█░`), ~10 символів.
- Для loop: `time % duration`; мітка `loop` в кінці рядка.
- Вимкнений трек (enabled = false): рядок відображається з нижченою яскравістю / `[paused]`.
- Якщо треків немає — overlay прихований.

**Реалізація:**
- `PIXI.Text` або `PIXI.BitmapText` у лівому нижньому куті stage.
- Оновлення тексту прямо в `tickerFn` — жодного Vue.
- Шрифт моноширинний, дрібний (~11–12px), напівпрозорий фон.

---

## 10. Плейлист система (внутрішня)

`trackPlaylists: Record<number, TrackQueueEntry[]>` — майстер-список намірів.

- Зберігається окремо від Spine (Spine видаляє entries після завершення).
- При натисканні **Play** (не Resume) — повна послідовність відновлюється в Spine для кожного увімкненого треку.
- Завдяки цьому послідовність A→B→C можна запускати знову без повторного вибору.
